DataStore.swift functions as the application's centralized state management and persistence engine. The file manages reading session creation and completion, page-turn recording, achievement detection, integrity validation, corruption recovery, and JSON serialization to disk. SessionCoordinator.swift serves as the application's active-session orchestration layer. It manages the lifecycle of the currently active reading session by coordinating session startup, page navigation, session restoration, elapsed-time tracking, and session termination. The file maintains the authoritative runtime state for the user's active book, active session, current page position, and reading duration while delegating permanent storage responsibilities to DataStore. It uses observable published properties to keep the user interface synchronized with session activity in real time. Architecturally, this file acts as a controller between the reading interface and the persistence layer, ensuring that live reading behavior is translated into structured session data. Combined Relationship of the Three Files Together, these three files form the application's core reading-state architecture. Book.swift defines the data models, DataStore.swift manages and persists instances of those models, and SessionCoordinator.swift controls the live runtime behavior that generates and updates that data. Reading sessions originate in SessionCoordinator, are recorded and persisted through DataStore, and are ultimately stored using the structures defined in Book.swift. As a result, these files collectively provide the data model layer, state management layer, persistence layer, and active session management layer that support the application's reading-tracking functionality.
